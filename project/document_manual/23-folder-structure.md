# অধ্যায় ২৩: ফোল্ডার স্ট্রাকচার

## ২৩.১ উদ্দেশ্য
প্রতিটি শিক্ষার্থীর ডকুমেন্ট একটি সুসংগঠিত ফোল্ডার কাঠামোতে রাখা।

## ২৩.২ স্ট্যান্ডার্ড ফোল্ডার কাঠামো
```
Hangeul-Students/
└── [Intake]/                (যেমন 2026-Spring)
    └── [Program]/           (KLP / EAP / Bachelor / Master / PhD)
        └── [StudentID]_[Name]/
            ├── 01-Identity/          (Passport, NID, BirthCert, Photo)
            ├── 02-Academic/          (Certificates, Transcripts)
            ├── 03-Language/          (IELTS)
            ├── 04-SOP-LOR/           (SOP, Recommendation, Proposal)
            ├── 05-Financial/         (BankStatement, Solvency, eTIN, TradeLicense)
            ├── 06-Family/            (FamilyRelation, Guardian docs)
            ├── 07-Translated-Notarized/  (অনূদিত/নোটারাইজড/অ্যাটেস্টেড)
            ├── 08-Payment/           (PaymentProof, Receipt)
            └── 09-University-Submission/  (আবেদন, অফার, ভিসা)
```

## ২৩.৩ নিয়ম
- প্রতিটি শিক্ষার্থীর জন্য একটি রুট ফোল্ডার (StudentID_Name)।
- ডকুমেন্ট সঠিক সাব-ফোল্ডারে (এলোমেলো নয়)।
- মূল ও প্রক্রিয়াকৃত (অনূদিত/নোটারাইজড) আলাদা।
- অ্যাক্সেস শুধু অনুমোদিত টিম (গোপনীয়তা)।

## ২৩.৪ Red Flags / সাধারণ ভুল
- ⚠️ সব ফাইল এক ফোল্ডারে গাদা করা।
- ⚠️ ভুল সাব-ফোল্ডার।
- ⚠️ ডুপ্লিকেট/পুরোনো ভার্সন মিশ্রিত।

## ২৩.৬ এসকালেশন / FAQ / অনুশীলন
- **এসকালেশন:** কাঠামো লঙ্ঘন → ম্যানেজার।
- **FAQ:** "একই নামে দুই শিক্ষার্থী?" → StudentID পার্থক্য নিশ্চিত করে।
- **অনুশীলন:** একজন নমুনা শিক্ষার্থীর সম্পূর্ণ ফোল্ডার কাঠামো তৈরি করুন।
